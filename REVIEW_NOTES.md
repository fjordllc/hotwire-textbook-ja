# レビューノート

執筆・レビュー時に確認する観点をまとめます。

## 技術確認

- 公式ドキュメントの URL が実在するか。
- Rails の対象バージョンでコードが動くか。
- Turbo / Stimulus / Hotwire Native の最新仕様とずれていないか。
- ハンズオンの手順を初期状態から最後まで再現できるか。

## 読みやすさ

- Rails 学習済みの読者にとって、既知の内容を説明しすぎていないか。
- 逆に、Hotwire 特有の考え方を省略しすぎていないか。
- 各章で「なぜそうするのか」が説明されているか。
- ハンズオンの途中で読者が状態を確認できるか。
