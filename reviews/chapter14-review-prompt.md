# 重点的に見てほしい観点（主張の正確さと判断軸の妥当性）
1. 技術的主張が正確か。要検証:
   - frame 内リンクは既定でいちばん内側の frame を差し替え、data-turbo-frame で対象を変えられる、という説明
   - frame 内遷移は URL を変えない、data-turbo-action="advance" で frame 遷移を URL も進める visit に格上げできる、という記述（link_to の data: { turbo_frame:, turbo_action: "advance" } 表記の正否）
   - 「1 つの frame は 1 か所しか差し替えられない、複数箇所同時更新は Turbo Streams」という切り分けが正しいか
2. controller 分岐の議論が妥当か。「同じビューに frame を置けば Turbo が必要な frame を取り出すので分岐は不要」という主張が、Rails/Turbo の設計思想と一致するか。turbo_frame_request? などの存在に触れずに済ませている点は許容できるか。
3. 判断軸の妥当性。Streams へ切り替える / 通常遷移へ戻す、の2つの判断基準が実務的に妥当か。第7部・第36章（アンチパターン）と矛盾しないか。
4. 誤った一般化がないか（frame を全否定していないか、逆に万能視していないか）。「できるか」でなく「読みやすく保てるか」という基準の置き方。
5. 本文の質（STYLEGUIDE 準拠）。第5部 Turbo Streams への接続が自然か。
